# Suzy Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing transcript-search app into the premium Phase 1 Suzy product by wrapping the current brain with landing, auth, session persistence, and a polished member chat experience.

**Architecture:** Keep the current retrieval and answer-generation pipeline as the backend brain, then add a product layer on top with Supabase magic-link auth, persisted chat sessions/messages, a non-streaming product chat wrapper, and a premium UI that hides source mechanics. Keep welcome as the initial logged-in state, generate session titles server-side from the first user message, and defer all GHL/payment/revoke-access concerns to Phase 2.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Supabase Auth + Postgres, existing OpenRouter/Supabase search code, Vitest, React Testing Library, jsdom

## Notes for Implementers

- Keep the existing brain intact unless a tiny extraction is required to reuse it cleanly.
- Do not add GHL sync, revoke-access logic, payments, or admin systems.
- Do not add message editing, regeneration, or manual title editing in Phase 1.
- Keep starter prompts static copy in Phase 1 unless a real requirement emerges.
- Generate session titles synchronously on first-message save inside the server route unless implementation pressure proves a post-save helper is materially simpler.
